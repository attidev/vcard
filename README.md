# vCard QR 코드 생성기

회사 직원들의 연락처 정보가 포함된 vCard QR 코드를 일괄 생성하는 웹 애플리케이션입니다.

## 주요 기능

- 📊 **CSV 파일 업로드**: 직원 정보를 CSV 파일로 관리
- 🏢 **공통 정보 설정**: 회사명, 주소, 웹사이트 등 공통 정보 설정
- 📱 **vCard 형식**: 표준 vCard 3.0 형식으로 QR 코드 생성 (ZXing 호환)
- 📥 **개별/일괄 다운로드**: 각 QR 코드를 개별 다운로드하거나 ZIP 파일로 일괄 다운로드
- ✏️ **쉬운 관리**: CSV 파일 수정으로 추가/삭제 가능
- 🔄 **자동 최적화**: 데이터가 길 경우 자동으로 최소 필드만 포함하는 vCard로 축소

## 사용 방법

1. **CSV 템플릿 다운로드**
   - 웹페이지에서 "CSV 템플릿 다운로드" 버튼 클릭
   - 또는 `template.csv` 파일 사용

2. **CSV 파일 작성**
   - Excel 또는 텍스트 에디터로 CSV 파일 열기
   - 각 행에 직원 정보 입력:
     - 이름 (필수)
     - 직책
     - 이메일
     - 전화번호
     - 휴대폰
     - 부서

3. **QR 코드 생성**
   - CSV 파일 업로드 (드래그 앤 드롭 또는 파일 선택)
   - 회사명 등 공통 정보 입력
   - "QR 코드 생성" 버튼 클릭

4. **다운로드**
   - 개별 다운로드: 각 QR 코드 아래의 "다운로드" 버튼 클릭
   - 전체 다운로드: "전체 다운로드 (ZIP)" 버튼 클릭

## CSV 파일 형식

CSV 파일은 다음 컬럼을 포함해야 합니다:

- `이름` (필수)
- `직책` 또는 `직위`
- `이메일`
- `전화번호` 또는 `핸드폰`
- `휴대폰` 또는 `mobile`
- `부서`

컬럼명은 한글 또는 영문 모두 지원됩니다:
- 이름 / name / Name
- 직책 / position / Position / 직위
- 이메일 / email / Email
- 전화번호 / phone / Phone / 핸드폰
- 휴대폰 / mobile / Mobile
- 부서 / department / Department

## vCard 형식

생성되는 vCard는 ZXing 표준 형식을 따릅니다:

```
BEGIN:VCARD
VERSION:3.0
N:이름
ORG:회사명
TEL:전화번호
URL:웹사이트
EMAIL:이메일
ADR:주소
END:VCARD
```

## 기술 스택

- **HTML5 / CSS3 / JavaScript (Vanilla)**: 순수 JavaScript로 구현, 빌드 도구 불필요
- **QRCode.js v1.2.2**: QR 코드 생성 라이브러리
- **JSZip v3.10.1**: ZIP 파일 생성 라이브러리

## 의존성

프로젝트는 다음 두 개의 외부 라이브러리만 필요합니다:

1. **qrcode.min.js** - QR 코드 생성
   - 다운로드: https://cdn.jsdelivr.net/npm/qrcode@1.2.2/build/qrcode.min.js
   - 또는 CDN 자동 로드 지원

2. **jszip.min.js** - ZIP 파일 생성
   - 다운로드: https://cdn.jsdelivr.net/npm/jszip@3.10.1/dist/jszip.min.js
   - 또는 CDN 자동 로드 지원

**추가 의존성 없음**: Node.js, npm, 빌드 도구 등이 필요하지 않습니다.

## 제한사항 및 성능

### 최대 인원 수
- **명시적인 제한 없음**: 이론적으로는 수천 명까지 가능합니다
- **실제 권장 사항**:
  - **100명 이하**: 가장 빠르고 안정적
  - **100~500명**: 정상 작동하지만 시간이 걸릴 수 있음
  - **500명 이상**: 브라우저 메모리와 성능에 따라 느려질 수 있음

### 성능 고려사항
- 각 QR 코드 생성에 약 0.1~0.5초 소요 (데이터 길이에 따라 다름)
- 100명 생성 시 약 10~50초 소요
- 500명 생성 시 약 50~250초 소요
- 진행 상황이 화면에 표시됩니다

### 브라우저 메모리
- 각 QR 코드는 약 100KB의 메모리를 사용합니다
- 1000명 생성 시 약 100MB의 메모리 사용
- 대부분의 최신 브라우저에서 충분히 처리 가능합니다

## 주의사항

- CSV 파일은 UTF-8 인코딩을 사용해야 합니다
- 한 번에 많은 수의 QR 코드를 생성할 경우 시간이 걸릴 수 있습니다
- 생성된 QR 코드는 명함 인쇄에 적합한 해상도(300x300px)로 생성됩니다
- 데이터가 너무 길 경우 자동으로 최소 필드만 포함하는 vCard로 축소됩니다
- 많은 수의 QR 코드를 생성할 경우 브라우저가 느려질 수 있으니, 필요시 CSV 파일을 나눠서 생성하는 것을 권장합니다

## 문제 해결

### QR 코드가 생성되지 않는 경우
1. 브라우저 콘솔(F12)에서 오류 메시지 확인
2. 라이브러리 파일(`qrcode.min.js`, `jszip.min.js`)이 같은 폴더에 있는지 확인
3. 인터넷 연결 확인 (CDN 자동 로드 시)

### QR 코드를 스캔했을 때 정보가 표시되지 않는 경우
1. QR 코드 리더 앱이 vCard 형식을 지원하는지 확인
2. 생성된 QR 코드가 선명한지 확인
3. 데이터가 너무 길어서 축소되었을 수 있음 (콘솔 로그 확인)
